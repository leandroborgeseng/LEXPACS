# Internacionalização (i18n) — LEX PACS

Guia de tradução do produto: onde estão os textos, como adicionar idiomas e o que ainda falta migrar.

---

## Idiomas suportados

| Código  | Idioma             | Viewer OHIF | Portal paciente | Login clínico |
|---------|--------------------|-------------|-----------------|---------------|
| `pt-BR` | Português (Brasil) | Sim         | Sim             | Sim           |
| `en-US` | Inglês (EUA)       | Sim         | Sim             | Sim           |
| `es`    | Espanhol           | Sim         | Sim             | Sim           |

O OHIF também inclui outros idiomas upstream (fr, de, ja-JP…), mas **strings LEX** existem só nos três acima.

---

## Fonte única de traduções

```
locales/
├── README.md
├── pt-BR/
│   ├── LexPacs.json       → viewer (componentes LEX)
│   ├── Portal.json        → portal do paciente
│   └── ClinicalLogin.json → login clínico
├── en-US/
│   └── …
└── es/
    └── …
```

**Sincronizar após editar:**

```bash
./scripts/sync-locales.sh
```

Isso copia os JSON para:

- `ohif-viewer/platform/i18n/src/locales/{lang}/LexPacs.json`
- `lex-pacs-portal/static/locales/{lang}/`

## Como cada parte do sistema traduz

### Viewer OHIF (React)

- **Motor:** `@ohif/i18n` (i18next + react-i18next)
- **Config:** `ohif-viewer/platform/i18n/src/index.js`
- **Fallback:** `en-US`
- **Namespace LEX:** `LexPacs`

```tsx
import { useTranslation } from 'react-i18next';

const { t, i18n } = useTranslation('LexPacs');
t('report.title');
date.toLocaleString(i18n.language);
```

**Trocar idioma:** User Preferences → Language, ou `?lng=pt-BR` na URL. Persistido em `localStorage` (`i18nextLng`).

**Componentes LEX já migrados:**

| Arquivo | Namespace |
|---------|-----------|
| `PanelLexReport.tsx` | LexPacs |
| `PacsStatsCharts.tsx` | LexPacs |
| `WorkListViewBar.tsx` | LexPacs |
| `StudyListSettingsPopover.tsx` | LexPacs + AboutModal |
| `PacsSettingsModal.tsx` | LexPacs | Completo |
| `getPanelModule.tsx` | LexPacs |

**OHIF nativo (não LEX):** namespaces como `StudyList`, `Buttons`, `SidePanel`. O `pt-BR` upstream cobre só parte; colunas da worklist nova ainda estão em inglês fixo em `defaultColumns.tsx`.

### Portal do paciente e login clínico (vanilla JS)

- **Helper:** `lex-pacs-portal/static/i18n.js` (`LexI18n`)
- **JSON:** `/static/locales/{lang}/Portal.json` e `ClinicalLogin.json`

```html
<h1 data-i18n="login.title"></h1>
<script src="/static/i18n.js"></script>
```

```javascript
await LexI18n.init(['Portal']);
LexI18n.t('studies.viewStudy');
```

Compartilha `localStorage.i18nextLng` com o viewer.

### Backend FastAPI

**Ainda sem i18n.** Mensagens de `HTTPException(detail=…)` estão em português. O frontend exibe `detail` da API diretamente.

**Próximo passo sugerido:** retornar códigos (`error_code: "invalid_credentials"`) e traduzir no cliente via namespace `Errors`.

## Fluxo de trabalho para novo texto

1. Adicionar chave em `locales/pt-BR/`, `en-US/` e `es/` (mesmo namespace).
2. `./scripts/sync-locales.sh`
3. Viewer: `t('namespace.key')` no componente → `docker compose build ohif`
4. Portal: `data-i18n="key"` ou `LexI18n.t('key')` → rebuild portal ou reload estático
5. Validar os três idiomas: `?lng=en-US`, `?lng=es`, `?lng=pt-BR`

## Tema claro / escuro

- Chave compartilhada: `localStorage['lex-theme']` (`light` | `dark`)
- Viewer: botão sol/lua na worklist e no header do visualizador
- Portal e login clínico: botão no header (`theme.js`)
- Preferência persiste entre recarregamentos e sincroniza entre abas (evento `storage`)
- Padrão: escuro; sem valor salvo, respeita `prefers-color-scheme`
